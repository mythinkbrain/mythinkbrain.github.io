7. Sebuah jaringan memiliki Bandwith 512 kbps, akan mendownload file dengan ukuran 600 byte berapa waktu yang di butuhkan untuk mendownload file tersebut


Bandwith 512 Kbps (Kilobits/s) = 64 KBps (Kilobytes/s) = 65.536 Bytes/s

Kalau ukuran file = 600 Byte (Bentar, rasanya nggak mungkin 600 Bytes), maka lama download :

600 B : 65.536 B/s = 0,009... detik.

Kalau ukuran file = 600 Kilobyte (Ini yang peling mungkin), maka lama download :

600 KB : 64 KB/s = 9,375 detik.

13. jika kecepatan internet saya 4,5mbps dan total size file download saya 20gb, maka berapa lamakah waktu yang dibutuhkan untuk mendownload file tersebut?


Jawaban:

jika mengikuti logika maka jawabanya adalah 9,8763889 jam / 35555.5555556 detik

jika mengikuti soal maka jawabanyan adalah 4444.44444444 detik / 1,23455556 jam

Penjelasan:

mengikuti logika

4,5 mbps (megabit per second)

karna satuan dari provider itu Megabit maka kita harus rubah dahulu ke megabyte persecond

1 mbps (megabit persecond) = 0.125 megabyte (mb)

4,5 x 0.125 = 0.5625 megabyte per second

ukuran file 20gb

20gb = 20000mb

20000 : 0.5625 = 35555.5555556 detik

jika di rubah menjadi jam maka menjadi 9,8763889 jam

mengikuti soal

20gb = 20000mb

20000 : 4.5 = 4444.44444444 detik

atau 1,23455556 jam

26. pak danang mendownload sebuah vidio yang berukuran 50mb dengan kecepatan download internet 100kbps, berapa detikkah waktu yang dibutuhkan pak danang untuk mendownload vidio tersebut?


Bila dikonversikan: 1 mega byte = 1.024 kilo byte
maka 50 mega byte = 51.200 kilo byte
kecepatan download = 100 kilo byte per second
maka waktu yang dibutuhkan: ukuran file / kecepatan download = 51.200 / 100 =512 second.

Jadi waktu yang dibutuhkan pak danang adalah 512 detik (kira-kira 8 menit)

Kesimpulan
